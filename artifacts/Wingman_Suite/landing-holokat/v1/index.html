Wingman is a self-hosted suite for running AI agents, coordinating the work they do, and keeping humans in the loop across projects, apps, workflows, files, and teams.

See the stack Installer

A shared workspace for people and agents.

Think of it as a team operations layer for AI work. Autopilot can run Codex, Claude Code, Goose, Pi, and other agents. Flight Deck gives people a common surface for chat, tasks, docs, approvals, and project context. Tower provides the Nostr/NIP-98 authenticated backend, Postgres state, graph knowledge, storage, and encrypted sync when needed.

The suite, in practical order.

1. Autopilot

Runs agent sessions, pipelines, triggers, managed apps, subdomain routing, files, and NIP-98 agent APIs.

2. Flight Deck

The browser workspace for chat, tasks, docs, flows, approvals, scopes, and shared project coordination.

3. Tower

The backend authority: Nostr auth, Postgres, graph knowledge, storage, SuperBased-style encrypted records, and typed APIs.

4. Skills

Repeatable procedures agents can load so work moves from ad hoc prompts into reliable operating playbooks.

Built for orchestration, not solo chat.

01

Spin up focused workers.

Dispatch agents into the right repo, task, or pipeline with enough context to produce a tested result.

02

Keep the work visible.

Humans can follow chats, board state, docs, artifacts, approvals, and handoffs in Flight Deck.

03

Coordinate across people.

Teams can share scopes, projects, WApps, and workflows while each agent signs and acts with its own identity.
