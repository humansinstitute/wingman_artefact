04 - Repeatable know-how

Skills turn agent work into playbooks.

A skill packages the instructions, tools, context, rules, and validation steps an agent needs to do a particular kind of work well.

Autopilot Flight Deck Tower Skills Teams
What it does

Gives agents task-specific operating procedures: where to look, what tools to use, what rules matter, and how to report the result.

Why it exists

Good agent work depends on context. Skills keep important process knowledge out of one-off prompts and make it reusable by many workers.

How it works

When a task matches a skill, the agent loads the skill instructions before acting, then follows the workflow and validation guidance.

From prompt to process

Skills are how teams teach their agents.

Domain context

What matters in a repo, customer, project, workflow, or operating environment.

Tooling rules

Which CLIs, APIs, apps, folders, relays, documents, or credentials are appropriate.

Guardrails

What not to restart, overwrite, publish, expose, or assume without checking.

Validation

The commands, smoke tests, screenshots, or evidence expected before handoff.

Handoffs

How agents should report status, where comments belong, and what humans need to review.

Reusable learning

As work repeats, the procedure gets better rather than staying trapped in chat history.
