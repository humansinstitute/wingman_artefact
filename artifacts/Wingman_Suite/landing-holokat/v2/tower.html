03 - Backend authority

Tower keeps the shared state trustworthy.

It is the backend layer for workspace records, Nostr identity, NIP-98 signed APIs, Postgres data, storage, graph knowledge, and encrypted record sync.

Autopilot Flight Deck Tower Skills Teams
What it does

Stores and serves the shared workspace data: tasks, chats, docs, files, comments, groups, scopes, graph memory, and app records.

Why it exists

Agent work needs a system of record. Tower gives each user, app, and agent an identity and a backend contract instead of loose local files.

How it works

Agents and apps call Tower with Nostr/NIP-98 signed requests. Tower validates identity, applies access rules, and writes durable state.

State and memory

Tower is where shared work becomes durable.

Postgres records

Typed APIs for workspace data such as chats, tasks, docs, comments, files, grants, and scopes.

Nostr auth

People, agents, apps, and workspace keys can sign actions instead of relying on a single shared account.

Graph knowledge

Named people, projects, sessions, apps, and prior work can become queryable memory.

Storage

Files, images, audio, generated outputs, and artifacts can be stored with workspace metadata.

Groups

Access can be scoped to teams and workspaces rather than being globally open.

Encrypted sync

SuperBased-style encrypted records remain available where compatibility or privacy calls for it.
