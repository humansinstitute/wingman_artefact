01 - Agent runtime

Autopilot is where the agents actually run.

It is the engine room: sessions, pipelines, triggers, managed apps, hosted links, files, and API access for agents that need to do work rather than just answer a prompt.

Autopilot Flight Deck Tower Skills Teams
What it does

Starts and manages agents such as Codex, Claude Code, Goose, Pi, or other workers. It gives them sessions, working directories, environment, files, and lifecycle controls.

Why it exists

Most AI tools are trapped in one chat. Autopilot lets an agent become a repeatable worker that can run in a repo, launch tools, host an app, or execute a workflow.

How it works

A human or workflow dispatches a session or pipeline. Autopilot starts the worker, attaches the right context, tracks status, and exposes results back to the suite.

Autopilot handles

The practical machinery around agent work.

Sessions

Long-lived or focused agent runs with a goal, working directory, transcript, and state.

Pipelines

Repeatable multi-step workflows for research, implementation, review, publishing, or data processing.

Managed apps

Small WApps, prototypes, internal tools, and hosted subdomain routes that agents or teams can use.

Triggers

Scheduled, event-driven, or manual starts for sessions and pipelines.

Files

Inputs, outputs, artifacts, screenshots, generated documents, and handoff material.

Signed APIs

NIP-98 style access so agents can call platform services with an identity instead of a shared secret.
